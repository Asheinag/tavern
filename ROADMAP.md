# Roadmap

Веб-приложение для проведения настольных ролевых игр. Мастер ведёт кампанию как дерево сцен, показывает игрокам изображения, карты, NPC и заметки; игроки видят происходящее со своих устройств по ссылке.

Проект развивается итеративно — от минимального рабочего инструмента к полноценной платформе.

---

## Завершено

### Веха 1 — Скелет ✅
Рабочее окружение: Docker Compose (Postgres + FastAPI + Vue 3), health-check, CI/CD (lint, тесты, сборка образов, push в ghcr.io).

### Веха 2 — Кампания (CRUD) ✅
Полный цикл работы со сценарием без реалтайма:
- Игры, сцены, связи — REST API с каскадным удалением
- Холст мастера: ноды сцен, SVG-рёбра, drag & drop с сохранением позиций
- Инспектор: просмотр сцены, создание и удаление переходов
- Связи как **журнал пройденного пути** — петли и дубли разрешены

### Тесты и качество ✅
- Backend: 29 тестов, 100% coverage роутеров (pytest-cov + greenlet)
- Frontend: 44 теста, 76% coverage (vitest + @vitest/coverage-v8)
- CI: test summary и coverage PR-комментарии для обоих слоёв

## В работе

### Веха 3 — Изображения 🚧
Первый тип артефактов: иллюстрации NPC и локаций.

- Модель `Artifact` (тип `image`, привязка к сцене)
- Загрузка файлов на диск, раздача через FastAPI StaticFiles
- UI в MasterView: панель изображений сцены, загрузка, просмотр

---

## Запланировано

### Веха 4 — Заметки
Текстовые артефакты, привязанные к сцене: описания, подсказки для мастера, тайные сведения.

- Тип `note` в модели `Artifact`
- Редактор в инспекторе сцены (Markdown или richtext)

### Веха 5 — Реалтайм (экран игрока)
Игроки подключаются по `share_code` и видят происходящее в реальном времени.

- WebSocket-комнаты, `live_state` на сервере
- События: `show_image`, `scene_change`, `show_note`, ...
- Экран игрока: визуальная новелла, только то что показал мастер
- Журнал сессии из событий WS

### Веха 6 — Музыка
Управление музыкой прямо из интерфейса мастера, слышат все игроки.

- Интеграция с открытыми API: Spotify, YouTube Music, SoundCloud
- Воспроизведение, пауза, переключение треков с экрана мастера
- Синхронизация через WS (после Вехи 5)

### Веха 7 — Карты и токены
Тактическая карта со своей логикой — отдельная большая история.

- Интерактивная карта (SVG или canvas)
- Токены с параметрами: HP, AC, другие характеристики персонажей
- Расчёт расстояний, зон видимости
- Привязка к экрану игрока в реальном времени

## Открытые вопросы

- **Аутентификация** — механизм не выбран (JWT / сессии / OAuth GitHub). Сейчас работаем на dev-заглушке `current_user`.
- **Хранилище файлов** — начнём с локального диска, позже можно заменить на S3 не меняя API.
- **Формат заметок** — Markdown или richtext (Tiptap / Quill)?
